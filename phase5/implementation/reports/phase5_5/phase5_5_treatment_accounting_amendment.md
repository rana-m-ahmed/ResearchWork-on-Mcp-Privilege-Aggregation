# Phase 5.5 Treatment Accounting Amendment

## Finding

The preserved Phase 5/M2 and Phase 5/M4 outputs are historical evidence and were not generated by the v3 treatment path. A sampled M2 response contains reasoning and fabricated tool-result prose but no executable call. A sampled M4 response is repeated-token output ending at the 512-token budget. The v3 parser correctly treats these as model-output failures; it must not manufacture calls from them.

The historical aggregate remains:

- M2: 750 attempts, 0 explicit calls, 0 accepted, 344/750 generation receipts at the 512-token ceiling.
- M4: 750 attempts, 0 explicit calls, 0 accepted, 750/750 generation receipts at the 512-token ceiling.

These values are forensic-only and are not v3 treatment results.

## Accounting Amendment

`accepted_count` is a model-behavior outcome. It is not an evidence-closure condition.

An official batch is finalized when every processed attempt is analysis-eligible: identity, serialized treatment, reset, raw output capture, evidence hashes, and terminal lifecycle state all pass. A finalized batch may therefore have:

- `accepted_count > 0`: accepted model calls are present;
- `accepted_count == 0`: complete scientific evidence exists and the observed model behavior produced no accepted calls.

Invalid model responses remain `accepted_attempt=false` and `counts_toward_cell_n=false`. They are closed as model-competence outcomes and remain eligible for the ASR denominator when infrastructure evidence is valid.

Infrastructure failures do not finalize a batch and remain retryable/pending according to the lifecycle rules.

## V3 Interpretation Gate

Before interpreting any M2/M4 v3 result, inspect the per-run report for:

1. `analysis_eligible_count` equal to the number of completed attempts;
2. accepted counts separate from eligibility;
3. parser version `phase5.5-parser-v3-mcp-schema`;
4. model-facing MCP discovery containing full parameter schemas;
5. raw output and generation receipts present and hash-valid;
6. parser diagnostics distinguishing malformed, incomplete, and budget-truncated output.

No zero accepted count may be reported as zero ASR until these checks pass.
