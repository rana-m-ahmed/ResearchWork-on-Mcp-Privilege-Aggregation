# Phase 5.5 Evidence Namespace Remediation

## Scope

Prevent fresh official/pretrial campaigns from colliding with immutable historical attempt evidence when attempt numbering restarts at `A000`.

## Frozen inputs

- Phase 5 and Phase 5.5 frozen datasets, model identities, parser rules, and trial settings remain unchanged.
- Existing evidence under `phase5_5/evidence/attempts/` remains immutable.

## Acceptance criteria

- A new campaign writes attempts under `phase5_5/evidence/attempts/<run_id>/<attempt_id>`.
- Attempt IDs and lineage semantics remain unchanged.
- Resume validates the physical directory recorded by lineage, including portable old-checkout paths.
- Same-run duplicate writes still fail closed through the existing immutable writer.
- The change is applied to all model branches through the shared runtime source and source freezes.
