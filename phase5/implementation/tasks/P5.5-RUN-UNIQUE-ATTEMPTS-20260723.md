# Phase 5.5 Run-Unique Attempt Identity

## Scope

Eliminate cross-campaign attempt-ID collisions without weakening append-only lineage validation or changing frozen scientific inputs.

## Frozen inputs

- Phase 4 and Phase 4.5 remain untouched.
- Dataset, queue partition, model identity, parser, grader, trial count, and batch identifiers remain unchanged.
- Existing evidence and lineage remain immutable.

## Acceptance criteria

- New attempt IDs use the campaign run session token, not the reusable batch partition token.
- Attempt ID grammar and within-run parent chains remain unchanged.
- Global duplicate-attempt rejection remains fail-closed.
- Source-freeze generation stores a canonical 40-character commit SHA for full, short, and symbolic Git refs.
- Shared behavior is verified and propagated to all four Phase 5.5 model branches.
